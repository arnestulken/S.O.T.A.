### Events ###
## Syntax ##
# Ein Event beginnt immer mit einer Tilde(~), danach die Koordinaten zwischen denen das Event ausgeführt wird (TriggerArea).
# Nun folgt der Name des Events(Type).
# Jetzt folgen die Paramter, diese sind von Typ zu Typ unterschiedlich.
# Bei Parametern sind auch "Texte mit Leerzeichen" erlaubt.
#
# TriggerArea:
#    x1.y1;x2.y2
#    x.y
#    Die TriggerArea ist die Fläche zwischen dem Punkt1(x1,y1) und dem Punkt2(x2, y2) oder nur der Punkt(x, y)
# Type:
#    display, checkpoint, end, style, teleport, set_condition, if_condition
#
# Mögliche Parameter:
#    display:
#        message (string) HTML ist erlaubt
#        dauer der Anzeige in ms (int)
#    checkpoint:
#        Koordinaten des Checkpoints (x.y)
#        Soll eine nachricht angezeigt werden (boolean)
#    end:
#        typ (finish)
#    style:
#        background (url, htmlColor)
#        foreground (htmlColor) nur im Editor modus verfügbar
#    teleport:
#        Koordinaten des Zieles (x.y)
#    set_block:
#        zu setzende Char
#        Koordinaten des zu setzenden Blocks (x.y)
#    set_condition:
#        key der Bedingung (string)
#        wert der Bedingung (string)
#    if_condition:
#        key der Bedingung (string)
#        sollwert der Bedingung (string)
#        hinter dem "->" ein event schreiben
#
## Beispiele ##
#~7.6 display "Welcome to\n <b>THE CAVE MAP</b>\n by Maik Müller" 1000
#~56.14;48.14 checkpoint 55.14 true
#~8.6 setblock | 7.6
#~6.6 style http://www.example.com/image.jpg #ffffff
#~8.6 set_condition schlüssel1 true
#~15.6 if_condition schlüssel1 true -> setblock | 7.6
#~115.32 end finish
#
